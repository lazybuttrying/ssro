# 02. Outputs walkthrough

This tutorial explains how to read the artifacts generated by SSRO.

## 1. `research_brief.json`
This is the governance-layer starting point.

It tells you:
- the title of the project
- the research question
- the current stage
- primary goals
- next steps

Read this first if you want to understand the intended workflow.

## 2. `literature_map.json`
This is the literature and theory layer output.

It includes:
- core concepts
- possible mechanisms
- hypothesis candidates

This is where the project starts to look like social science rather than just data processing.

## 3. `cleaned_data.csv`
This is the output of the data construction stage.

It shows:
- the cleaned starter dataset
- the unit of observation
- the variables before measurement rules are applied

## 4. `measured_data.csv`
This is the measurement stage output.

It adds variables such as:
- `education_score`
- `high_skill_signal`
- `routine_signal`
- `ai_exposure_score`
- `high_skill_job`

This file is important because it shows how abstract concepts become operational variables.

## 5. `codebook.json`
This explains what the measured variables mean.

Read this together with:
- `docs/measurement.md`

## 6. `analysis_results.json`
This is the main analysis output.

It contains:
- descriptive summaries
- a simple wage gap statistic
- interpretation notes
- suggested causal designs

This is still descriptive, not causal.

## 7. `figure_wage_by_ai_related.png`
This is a basic exported figure.

Its purpose is not to be publication-quality yet.
It is included to show that the repository can generate visible research artifacts, not only JSON files.

## 8. `reproducibility_report.json`
This is part of the audit layer.

It shows that the measured dataset can be checked for deterministic reproduction under the current code path.

## 9. `robustness_report.json`
This checks whether results move when the high-skill threshold changes.

This is a minimal sensitivity check.

## 10. `bias_validity_report.json`
This lists measurement and validity concerns.

This file matters because social science research is not only about running code. It is also about documenting what could be wrong.

## 11. `provenance_log.json`
This stores workflow history.

It records:
- what happened
- in what order
- with what key result

This is what makes SSRO look like a research system instead of a single assistant response.
